Security

How you use the system can affect the security of this information. You should be aware of your security options.

Password

Initially, your system administrator set a password for you to use Intranet. You should change your password before starting to use system to prevent unauthorized access to your information. SCM can not show you your password, but either SCM or your system administrator will be able to reset your password in case you have forgotten it. We recommend that you use one of your common passwords for accessing Intranet.

Cookies

A cookie is a data file that a web site writes and sends to your computer when you visit the site. SCM uses a cookie as a convenience to you. You do not need to accept them. If your browser is set to disable cookies, you might have to log back in to SCM when you return from another web site.

Our cookie file contains an encrypted version of your username. This information is used to identify who you are when you make a request to Intranet. If you allow your browser to accept cookies, a cookie is set and trusted when you return to SCM after visiting an external site. After you log out of SCM, the cookie is deleted.

SSL connection

The SCM uses SSL (Secure Sockets Layer) technology for transmitting private login information over the Internet. SSL works by encrypting data that is transmitted over the SSL connection. Both Mozilla Firefox and Internet Explorer support SSL. When you sign in to SCM, your username, and password are encrypted and then sent over the system using an SSL connection. No one can read or access the data that is being transmitted.

After you sign in and leave the encrypted connection, the system keeps track of who you are by using a computer-generated key. The key contains an encrypted version of your username and password.

If you have questions about the system security, please contact us.
